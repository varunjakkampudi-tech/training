Overview

Linux runs most servers, containers, and cloud infrastructure. Administration means managing files, users, processes, services, networking, storage, and security from the command line.

ConceptMeaning
KernelCore that manages hardware, processes, memory
ShellCommand interpreter (bash, zsh)
DistroKernel + tools (Ubuntu/Debian, RHEL/CentOS, Alpine)
Everything is a fileDevices, sockets, processes exposed as files
Interview importance: permissions, processes/systemd, networking, log/troubleshooting, and shell scripting are the most-asked Linux admin topics.

File & Directory Commands

Navigation & manipulation
ls -lah          # list all, long, human sizes
cd /var/log; pwd
cp -r src dst      # recursive copy
mv a b            # move/rename
rm -rf dir        # ⚠️ recursive force delete
mkdir -p a/b/c    # create nested dirs
ln -s target link # symbolic link
find /var -name "*.log" -mtime +7 -delete # old logs
Viewing files
cat file        # whole file
less file       # paged viewer
head -n 20 file  tail -f app.log  # follow live logs
wc -l file       # line count
Danger: rm -rf / or a stray space (rm -rf / tmp) can wipe the system. Double-check paths; prefer trash-cli where possible.

Permissions & Ownership

Reading ls -l output
-rwxr-xr--  1 alice devs  4096 Jul 21 file
 │└┬┘└┬┘└┬┘     owner group
 │ │  │  └ others: r--  (read)
 │ │  └── group:  r-x  (read, execute)
 │ └───── owner:  rwx  (read, write, execute)
 └─────── type: - file, d dir, l link
chmod / chown / umask
chmod 755 script.sh   # rwx r-x r-x (owner all, others read+exec)
chmod u+x,g-w file    # symbolic
chown alice:devs file # owner:group
umask 022            # default: new files 644, dirs 755
OctalPerm
7rwx
6rw-
5r-x
4r--
Special bits: setuid (4xxx) runs as owner, setgid (2xxx) inherits group, sticky (1xxx, on /tmp) only owner can delete their files.

Processes & Signals

Inspect & control
ps aux              # all processes
top / htop          # live resource view
kill -9 <pid>       # SIGKILL (force)
kill -15 <pid>      # SIGTERM (graceful)
pkill -f node       # by name/pattern
nice -n 10 cmd;  renice -n 5 -p <pid> # priority
jobs;  bg;  fg;  cmd &   # job control
nohup cmd &         # survive logout
SignalMeaning
SIGTERM (15)Ask to stop gracefully
SIGKILL (9)Force kill (can't be caught)
SIGHUP (1)Reload config
SIGINT (2)Ctrl+C interrupt

systemd & Services

Manage services
sudo systemctl start|stop|restart nginx
sudo systemctl enable nginx    # start on boot
systemctl status nginx
systemctl list-units --type=service --state=running
journalctl -u nginx -f        # follow service logs
Custom unit: /etc/systemd/system/app.service
[Unit]
Description=My Node App
After=network.target

[Service]
ExecStart=/usr/bin/node /srv/app/server.js
Restart=always
User=app
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
After editing units: sudo systemctl daemon-reload then enable --now app. Use Restart=always for crash recovery.

Disk & Storage

Space & mounts
df -h              # disk free per filesystem
du -sh /var/*      # dir sizes (find space hogs)
lsblk;  blkid       # block devices / UUIDs
mount /dev/sdb1 /mnt
sudo mkfs.ext4 /dev/sdb1   # format
# persist mounts in /etc/fstab (by UUID)
Full disk? df -h shows which FS is full, du -sh /var/* | sort -h finds the hog. Watch out for large logs, /tmp, and deleted-but-open files (lsof | grep deleted).

Logs & journald

Where to look
journalctl -xe            # recent errors
journalctl -u ssh --since "1 hour ago"
tail -f /var/log/syslog     # Debian
tail -f /var/log/messages   # RHEL
dmesg -T                   # kernel ring buffer
Rotate logs with logrotate (/etc/logrotate.d/) so they don't fill the disk.

Text Processing (grep/sed/awk)

The power trio + pipes
grep -rin "error" /var/log   # recursive, case-insensitive, line #
sed -i 's/old/new/g' file    # in-place replace
awk '{print $1, $9}' access.log  # columns

# pipeline: top 10 IPs hitting the server
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head

Shell Scripting (bash)

backup.sh
#!/usr/bin/env bash
set -euo pipefail            # exit on error/unset/pipe fail

SRC="/srv/app"
DEST="/backups"
DATE=$(date +%F)

if [[ ! -d "$DEST" ]]; then mkdir -p "$DEST"; fi

tar -czf "$DEST/app-$DATE.tar.gz" "$SRC"
for f in "$DEST"/*.tar.gz; do
  echo "backup: $f"
done
echo "Done ✓"
Always start scripts with set -euo pipefail, quote variables ("$var"), and check with shellcheck.

Cron & Scheduling

crontab -e
# ┌ min ┌ hour ┌ day ┌ month ┌ weekday
0 2 * * *   /srv/backup.sh          # every day 02:00
*/5 * * * *  /srv/health.sh          # every 5 min
0 0 * * 0   /srv/weekly.sh          # Sunday midnight
Alternatives: systemd timers (better logging/dependencies), at for one-off jobs, anacron for machines not always on.

SSH & Remote Access

Key-based login & transfer
ssh-keygen -t ed25519          # generate key pair
ssh-copy-id user@host           # install public key
ssh user@host -p 2222
scp file user@host:/path        # copy over ssh
rsync -avz src/ user@host:/dst  # efficient sync
Harden SSH (/etc/ssh/sshd_config): disable root login (PermitRootLogin no), disable password auth (keys only), change default port, and use fail2ban.

Firewall & Hardening

ufw / firewalld
sudo ufw allow 22/tcp;  sudo ufw allow 443/tcp
sudo ufw enable;  sudo ufw status
# RHEL: firewall-cmd --add-service=https --permanent; --reload
  • Least privilege: only open needed ports; deny by default.
  • Keep packages patched (unattended-upgrades).
  • Use fail2ban against brute force; SELinux/AppArmor for MAC.
  • Least-sudo; audit with auditd.

Monitoring & Troubleshooting

Resource inspection
top / htop       # CPU/mem live
free -h          # memory
vmstat 1;  iostat  # CPU/IO
uptime;  w       # load average, who's on
lsof -i :8080     # what's using a port
strace -p <pid>    # syscalls of a process
Load average (1/5/15 min) > number of CPU cores = overloaded. Correlate with top (CPU vs IO wait) to find the cause.

Scenario-Based Questions

Scenario 1 — Disk is 100% full Storage

Q: An app crashes; the server disk is full. How do you find & fix it?

Show approach

df -h to find the full FS, du -sh /* | sort -h (then drill down) to find the hog — usually logs. Truncate/rotate logs, remove old files, check deleted-but-open files with lsof | grep deleted (restart the holder). Add logrotate + alerts.

Scenario 2 — Service won't start systemd

Q: systemctl start app fails.

Show approach

systemctl status app and journalctl -u app -e for the error. Common causes: wrong ExecStart path, permission/user issue, port already in use (ss -tulpn), missing env/config. Fix, daemon-reload, restart.

Scenario 3 — High load / slow server Performance

Q: The server is sluggish; load average is high.

Show approach

uptime/top — is it CPU, memory, or IO wait? free -h for swap thrash, iostat for disk, ps to find the hog. Kill/renice the offender, add resources, or fix the root cause (runaway process, memory leak).

Scenario 4 — Can't reach a service on a port Networking

Q: A client can't connect to your app on :8080.

Show approach

Is it listening? ss -tulpn | grep 8080. Bound to 0.0.0.0 or just localhost? Firewall open? ufw status. Test locally (curl localhost:8080) then remotely (nc -zv host 8080). Check security groups if cloud.

Top Interview Questions

Explain Linux file permissions.

Each file has read/write/execute for owner, group, others (e.g. rwxr-xr-- = 754). Set with chmod (octal or symbolic); ownership with chown.

SIGTERM vs SIGKILL?

SIGTERM (15) asks a process to stop gracefully (can clean up); SIGKILL (9) forcibly terminates it and can't be caught. Try TERM first.

What is systemd?

The init system and service manager: starts/stops services (units), manages dependencies, boot, logging (journald), and timers. Managed via systemctl.

How do you find what's using a port?

ss -tulpn | grep :PORT (or lsof -i :PORT) shows the listening process and PID.

How do you troubleshoot a full disk?

df -h finds the full filesystem; du -sh drills into the biggest dirs; check logs, /tmp, and deleted-open files via lsof. Rotate/clean and alert.

What is the difference between a hard link and a symbolic link?

A hard link is another name for the same inode (same data, same FS). A symlink is a pointer to a path (can cross filesystems, breaks if the target moves).

What does chmod 755 mean?

Owner rwx (7), group r-x (5), others r-x (5) — common for executables/scripts and directories.

How do you schedule a task?

crontab -e for cron jobs, or a systemd timer for better logging/dependency handling; at for one-offs.

What is load average?

Average number of processes runnable/uninterruptible over 1/5/15 minutes. Above the CPU core count indicates overload; distinguish CPU vs IO wait.

How do you run a process in the background / after logout?

cmd & backgrounds it; nohup cmd & or a systemd service survives logout; disown detaches a job.

What is the difference between apt and dnf?

Package managers for different families: apt (Debian/Ubuntu, .deb) and dnf/yum (RHEL/Fedora, .rpm). Both install/update/remove and resolve dependencies.

How do you view live logs?

tail -f /var/log/... or journalctl -u service -f for systemd-managed services.

What is a runlevel / target?

systemd "targets" replace SysV runlevels — e.g. multi-user.target (server) vs graphical.target (desktop). Set with systemctl set-default.

How do you secure an SSH server?

Key-only auth (disable passwords), PermitRootLogin no, non-default port, firewall, and fail2ban against brute force.

What does set -euo pipefail do?

In bash: -e exit on error, -u error on unset variables, -o pipefail fail if any command in a pipe fails — making scripts safer.
