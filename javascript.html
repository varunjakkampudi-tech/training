Overview

JavaScript is a high-level, single-threaded, dynamically-typed language that runs in browsers and on servers (Node.js). It's the language of the web — the only one browsers natively execute.

Why companies use it

  • Ubiquitous — runs on every browser, server (Node), mobile (React Native), and desktop (Electron).
  • Full-stack — one language for frontend and backend.
  • Huge ecosystem — npm is the largest package registry in the world.
  • Event-driven & non-blocking — great for I/O-heavy apps.

How JavaScript Runs

JavaScript is interpreted and compiled. Modern engines (V8 in Chrome/Node, SpiderMonkey in Firefox) use JIT (Just-In-Time) compilation.

ComponentRole
Engine (V8)Parses, compiles, and executes JS. Contains the call stack + heap.
Call StackTracks function execution (LIFO). Single-threaded.
HeapUnstructured memory where objects live.
RuntimeEngine + Web APIs (browser) or libuv (Node) + callback queues + event loop.

Parsing → Compilation → Execution

  1. Parsing: source → tokens → Abstract Syntax Tree (AST).
  2. Compilation (JIT): the interpreter (Ignition) produces bytecode; the optimizing compiler (TurboFan) recompiles hot code to fast machine code.
  3. Execution: bytecode/machine code runs on the call stack.
Single-threaded call stack
function a() { b(); }
function b() { console.log("hi"); }
a();
// Stack: a() -> b() -> console.log -> pops back up
Single-threaded means one call stack — only one thing runs at a time. Concurrency comes from the runtime (Web APIs / libuv) + event loop, not extra threads in your JS.

Execution Context

Every time a function runs, JS creates an execution context — the environment holding its variables, scope chain, and this.

PhaseWhat happens
CreationSets up the Variable Environment: hoists declarations, creates scope chain, binds this.
ExecutionRuns code line by line, assigning values.
  • Global Execution Context — created once when the script loads.
  • Function Execution Context — created on every function call.
  • Contexts are pushed/popped on the call stack.
Creation vs execution phase
console.log(x); // undefined (hoisted, not yet assigned)
var x = 10;
console.log(x); // 10

Hoisting & TDZ

Hoisting moves declarations to the top of their scope during the creation phase. var is hoisted & initialized to undefined; let/const are hoisted but not initialized — creating the Temporal Dead Zone (TDZ).

var vs let/const
console.log(a); // undefined
var a = 1;

console.log(b); // ReferenceError: Cannot access 'b' before initialization (TDZ)
let b = 2;

foo();            // "works" — function declarations are fully hoisted
function foo() { console.log("works"); }
varletconst
ScopeFunctionBlockBlock
HoistedYes (=undefined)Yes (TDZ)Yes (TDZ)
ReassignableYesYesNo
RedeclarableYesNoNo
Common mistake: Using var in loops with async callbacks captures the final value. Use let for per-iteration binding.

Scope & Lexical Scope

Scope defines where variables are accessible. Lexical (static) scope means inner functions can access variables of their outer functions, determined by where code is written, not called.

Scope chain
const g = "global";
function outer() {
  const o = "outer";
  function inner() {
    const i = "inner";
    console.log(g, o, i); // inner sees all 3 via scope chain
  }
  inner();
}
  • Global scope — accessible everywhere.
  • Function scope — variables inside a function.
  • Block scope — let/const inside { }.

Closures

A closure is a function that remembers variables from its lexical scope even after the outer function has returned. One of the most-asked interview topics.

Counter closure
function createCounter() {
  let count = 0;              // private state
  return () => ++count;         // closes over count
}
const c = createCounter();
c(); c(); c();                 // 1, 2, 3

Real-world uses

  • Data privacy — module pattern, private variables.
  • Function factories — multiplyBy(2).
  • Memoization, debounce/throttle, event handlers, currying.
Classic bug: for (var i=0;i<3;i++) setTimeout(()=>console.log(i)) logs 3,3,3. Fix with let (block-scoped per iteration) or an IIFE.

🌍 Real-world

A rate limiter or once() helper uses a closure to remember how many times a function has run, without exposing that counter globally.

JS has 7 primitives (string, number, boolean, null, undefined, symbol, bigint) and objects. Primitives are immutable and copied by value; objects by reference.

this / call / apply / bind

this is determined by how a function is called, not where it's defined (except arrow functions, which inherit this lexically).

Call stylethis is
Regular function (non-strict)global object / undefined (strict)
Method obj.fn()the object obj
new Fn()the new instance
Arrow functioninherited from enclosing scope
call/apply/bindexplicitly set
Explicit binding
function greet(greeting) { return `${greeting}, ${this.name}`; }
const user = { name: "Aria" };

greet.call(user, "Hi");      // "Hi, Aria"  (args listed)
greet.apply(user, ["Hi"]);   // "Hi, Aria"  (args array)
const bound = greet.bind(user); // returns a new function
Interview classic: Losing this when passing a method as a callback. Fix with .bind(this) or an arrow function.

Prototypes & Inheritance

Every object has an internal [[Prototype]] link. Property lookups walk up the prototype chain until found or null. This is how JS does inheritance.

Prototype chain
function Animal(name) { this.name = name; }
Animal.prototype.speak = function() { return `${this.name} makes a sound`; };

const dog = new Animal("Rex");
dog.speak();                 // found on Animal.prototype
Object.getPrototypeOf(dog) === Animal.prototype; // true
Chain: dog → Animal.prototype → Object.prototype → null. Methods live on the prototype (shared), not per-instance (memory-efficient).

Classes & OOP

ES6 class is syntactic sugar over prototypes, with cleaner inheritance via extends/super.

Class inheritance
class Animal {
  constructor(name) { this.name = name; }
  speak() { return `${this.name} makes a sound`; }
}
class Dog extends Animal {
  speak() { return `${this.name} barks`; }
  #secret = 42;              // private field
  static create(n) { return new Dog(n); } // static method
}
new Dog("Rex").speak(); // "Rex barks"

Callbacks & Promises

Async work uses callbacks, promises, or async/await. Promises solve "callback hell" and provide chaining + error propagation.

Promise states & chaining
// A promise is pending -> fulfilled | rejected
const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 100);
});
p.then(v => v.toUpperCase())
 .then(v => console.log(v))
 .catch(err => console.error(err))
 .finally(() => console.log("cleanup"));
CombinatorBehavior
Promise.allAll succeed, or reject on first failure
Promise.allSettledWaits for all, returns status of each
Promise.raceSettles with the first to settle
Promise.anyFirst fulfilled, or AggregateError

Async / Await

async/await with error handling
async function getUser(id) {
  try {
    const res = await fetch(`/api/users/${id}`);
    if (!res.ok) throw new Error(res.status);
    return await res.json();
  } catch (e) {
    console.error("Failed:", e);
    throw e;
  }
}

// Run in parallel — don't await sequentially!
const [a, b] = await Promise.all([getUser(1), getUser(2)]);
Performance bug: awaiting in a loop runs requests sequentially. Collect promises and Promise.all them for parallelism.

The Event Loop

The event loop lets single-threaded JS handle async work. It moves callbacks from queues to the call stack when the stack is empty.

QueueExamplesPriority
MicrotaskPromises, queueMicrotask, MutationObserverHigh — drained fully after each task
MacrotasksetTimeout, setInterval, I/O, UI eventsLower — one per loop tick
Classic ordering question
console.log("1");
setTimeout(() => console.log("2"), 0);      // macrotask
Promise.resolve().then(() => console.log("3")); // microtask
console.log("4");
// Output: 1, 4, 3, 2  (sync -> microtasks -> macrotasks)
Key rule: After each macrotask, the event loop drains all microtasks before the next render/macrotask.

Fetch & AJAX

fetch with timeout (AbortController)
const ctrl = new AbortController();
const id = setTimeout(() => ctrl.abort(), 5000);

const res = await fetch("/api/data", {
  method: "POST",
  headers: { "Content-Type": "application/json" },

Functional Patterns

Currying, composition, memoization
// Currying
const add = a => b => c => a + b + c;
add(1)(2)(3); // 6

// Composition
const compose = (...fns) => x => fns.reduceRight((v, f) => f(v), x);

// Memoization
function memoize(fn) {
  const cache = new Map();
  return (...args) => {
    const k = JSON.stringify(args);
    if (!cache.has(k)) cache.set(k, fn(...args));
    return cache.get(k);
  };
}

Debounce & Throttle

Two of the most-asked machine-coding questions. Both limit how often a function runs.

Debounce

Runs after activity stops for N ms. Use for search-as-you-type, resize.

debounce
function debounce(fn, delay) {
  let t;
  return (...args) => {
    clearTimeout(t);
    t = setTimeout(() => fn(...args), delay);
  };
}

Throttle

Runs at most once per N ms. Use for scroll, mousemove.

throttle
function throttle(fn, limit) {
  let wait = false;
  return (...args) => {
    if (wait) return;
    fn(...args); wait = true;
    setTimeout(() => wait = false, limit);
  };
}

Deep vs Shallow Copy

Copying objects
// Shallow — nested refs shared
const shallow = { ...obj };
const shallow2 = Object.assign({}, obj);

// Deep — modern, handles Dates/Maps/cycles
const deep = structuredClone(obj);

// Deep (legacy) — loses functions/undefined/Dates
const deep2 = JSON.parse(JSON.stringify(obj));
Interview trap: Spread/Object.assign are shallow — mutating a nested object affects both copies.

Polyfills

A polyfill re-implements a modern API for older environments. Writing map, bind, and Promise polyfills is a common interview task.

Array.prototype.myMap & bind
Array.prototype.myMap = function(cb) {
  const out = [];
  for (let i = 0; i < this.length; i++)
    out.push(cb(this[i], i, this));
  return out;
};

Function.prototype.myBind = function(ctx, ...pre) {
  const fn = this;
  return (...args) => fn.apply(ctx, [...pre, ...args]);
};

Web Workers

Workers run scripts on a background thread, keeping the UI responsive during heavy computation. They communicate via postMessage.

main.js + worker.js
// main.js
const w = new Worker("worker.js");
w.postMessage({ n: 40 });
w.onmessage = (e) => console.log(e.data);

// worker.js
onmessage = (e) => postMessage(fib(e.data.n));
No DOM access in workers. Great for parsing, image processing, crypto, big loops.

Machine Coding Problems

Flatten a nested array Medium

Task: Implement flatten([1,[2,[3,[4]]]]) → [1,2,3,4] without Array.flat.

Show solution
Solution
function flatten(arr) {
  return arr.reduce((acc, v) =>
    Array.isArray(v) ? acc.concat(flatten(v)) : acc.concat(v), []);
}
Implement Promise.all Hard

Task: Recreate Promise.all.

Show solution
Solution
function promiseAll(proms) {
  return new Promise((resolve, reject) => {
    const res = []; let done = 0;
    proms.forEach((p, i) =>
      Promise.resolve(p).then(v => {
        res[i] = v;
        if (++done === proms.length) resolve(res);
      }, reject));
  });
}
Deep clone with cycles Hard

Task: Deep clone an object that may contain circular references.

Show solution
Solution
function deepClone(obj, seen = new WeakMap()) {
  if (obj === null || typeof obj !== "object") return obj;
  if (seen.has(obj)) return seen.get(obj);
  const copy = Array.isArray(obj) ? [] : {};
  seen.set(obj, copy);
  for (const k in obj) copy[k] = deepClone(obj[k], seen);
  return copy;
}
Other must-practice: debounce, throttle, curry, memoize, EventEmitter, deep equality, retry with backoff, promisify, group-by, LRU cache.

Scenario-Based Questions

Scenario 1 — Search-as-you-type is hammering the API Async

Q: Every keystroke fires an API call. How do you fix it?

Show approach

Debounce the handler (~300ms), cancel in-flight requests with AbortController, and cache results. Optionally add a minimum query length.

Scenario 2 — Output ordering Event Loop

Q: What logs and why? setTimeout(...,0) vs Promise.then vs sync?

Show approach

Synchronous code first, then all microtasks (promises), then macrotasks (timers). Explain the microtask/macrotask queues and that microtasks drain fully between macrotasks.

Scenario 3 — A memory leak in a SPA Memory

Q: Memory grows on every route change. How do you find and fix it?

Show approach

Take heap snapshots in DevTools, compare retained objects. Usual culprits: listeners/timers/subscriptions not cleaned up, detached DOM nodes held by closures. Add cleanup on unmount and use WeakMap for caches.

Production Problems

Numbers rounding wrong in checkout Bug

Report: Totals like $0.30 show as $0.30000000000000004.

Show fix

Cause: IEEE-754 floating point. Fix: work in integer cents, or use a decimal library, and format with toFixed(2) only for display.

Unhandled promise crashes the Node process Bug

Report: Server exits intermittently.

Show fix

Cause: Unhandled rejection. Fix: add .catch/try-catch to every async path, and a global process.on("unhandledRejection") handler for logging.

Top Interview Questions

The most-asked JavaScript questions with concise answers.

Explain closures with a use case.

A closure is a function bundled with references to its lexical scope, so it remembers outer variables after the outer function returns. Used for data privacy, function factories, memoization, and debounce/throttle.

Difference between == and ===?

== compares after type coercion; === compares value and type without coercion. Prefer ===.

Explain the event loop, microtasks and macrotasks.

JS is single-threaded. Async callbacks queue in microtask (promises) or macrotask (timers, I/O) queues. After the stack empties, the loop drains all microtasks, then one macrotask, and repeats.

How does this work?

Determined by call site: method → object, new → instance, standalone → global/undefined, arrow → lexical, or explicitly via call/apply/bind.

var vs let vs const?

var is function-scoped and hoisted to undefined; let/const are block-scoped with a TDZ; const can't be reassigned.

Explain prototypal inheritance.

Objects link to a prototype via [[Prototype]]; property lookups walk the chain until found or null. Classes are sugar over this.

Debounce vs throttle?

Debounce runs after activity stops for N ms (search input). Throttle runs at most once per N ms (scroll).

Shallow vs deep copy?

Shallow copies top-level only (spread/assign) — nested objects are shared. Deep copies everything (structuredClone).

Promise.all vs allSettled vs race vs any?

all: all or reject on first error. allSettled: waits for all with per-item status. race: first to settle. any: first fulfilled.

What is hoisting?

Declarations are processed in the creation phase: var/functions are hoisted (function fully), let/const are hoisted but uninitialized (TDZ).

call vs apply vs bind?

All set this. call takes comma args, apply takes an args array, bind returns a new bound function.

What is currying?

Transforming f(a,b,c) into f(a)(b)(c) — a series of unary functions, enabling partial application.

Explain the module pattern.

An IIFE (or module) returns a public API while keeping variables private via closure — encapsulation before ES modules.

What are Symbols?

Unique, immutable primitives used as non-colliding object keys and well-known hooks like Symbol.iterator.

Difference between null and undefined?

undefined = declared but unassigned; null = intentional empty value. typeof null is "object" (legacy bug).
