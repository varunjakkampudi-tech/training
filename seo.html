Overview

SEO (Search Engine Optimization) is the practice of improving a site so search engines can crawl, understand, and rank it, driving organic traffic.

PillarFocus
TechnicalCrawlability, indexing, speed, rendering, mobile
On-pageContent, keywords, meta tags, structure
Off-pageBacklinks, authority, reputation
Interview importance: For frontend/full-stack roles, technical SEO, Core Web Vitals, and JS rendering strategies (CSR vs SSR vs SSG) are the most-asked.

How Search Works

  1. Crawling — bots (Googlebot) discover URLs via links & sitemaps.
  2. Rendering — the page is rendered (including JS) to see final content.
  3. Indexing — content is analyzed and stored in the index.
  4. Ranking — for a query, results are ordered by hundreds of signals (relevance, authority, UX).
Key ranking signals: content relevance & quality (E-E-A-T), backlinks, Core Web Vitals/page experience, mobile-friendliness, HTTPS, freshness, and intent match.

Crawling & Indexing

Control indexing
<!-- Keep a page out of the index -->
<meta name="robots" content="noindex, follow">

// HTTP header alternative
X-Robots-Tag: noindex
DirectiveEffect
index/noindexAllow/deny indexing
follow/nofollowFollow/ignore links
Crawl budgetLimit for large sites — don't waste it on junk URLs
Gotcha: robots.txt Disallow blocks crawling but not indexing (a blocked URL can still appear). To remove from the index, use noindex and let it be crawled.

robots.txt & Sitemaps

robots.txt + sitemap.xml
# robots.txt
User-agent: *
Disallow: /admin/
Sitemap: https://site.com/sitemap.xml

<!-- sitemap.xml -->
<url>
  <loc>https://site.com/page</loc>
  <lastmod>2026-07-21</lastmod>
</url>
Submit sitemaps in Google Search Console; keep them fresh, under 50k URLs / 50MB, and list only canonical, indexable URLs.

Meta Tags

Essential head tags
<title>Best Running Shoes (2026) | Brand</title>       # ~50-60 chars
<meta name="description" content="...">            # ~150-160 chars
<link rel="canonical" href="https://site.com/shoes">
<meta name="viewport" content="width=device-width, initial-scale=1">

<!-- Open Graph / Twitter for social sharing -->
<meta property="og:title" content="...">
<meta property="og:image" content="...">
<meta name="twitter:card" content="summary_large_image">
Note: The meta keywords tag is ignored by Google. Title & description drive click-through, not direct ranking, but a compelling description boosts CTR.

Canonical & Duplicate Content

A canonical tag tells search engines the preferred URL when the same content is reachable via multiple URLs (params, http/https, trailing slash, pagination).

Self-referencing canonical
<link rel="canonical" href="https://site.com/product?id=1">
<!-- consolidates ranking signals to one URL -->
Handle: tracking params (?utm=), www vs non-www, http vs https, and pagination with canonicals + 301 redirects to avoid diluting authority.

Headings & Content

  • One <h1> per page describing the topic; logical h2/h3 hierarchy.
  • Semantic HTML (article, nav, main) aids understanding.
  • Descriptive alt text on images (also accessibility).
  • Search intent — match informational/navigational/transactional intent; write for humans first.
  • E-E-A-T — Experience, Expertise, Authoritativeness, Trust.

Structured Data (Schema.org)

JSON-LD markup helps engines understand content and unlocks rich results (stars, FAQs, breadcrumbs).

Product JSON-LD
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Running Shoe",
  "aggregateRating": { "@type": "AggregateRating", "ratingValue": "4.6", "reviewCount": "120" },
  "offers": { "@type": "Offer", "price": "99.00", "priceCurrency": "USD" }
}
</script>

Core Web Vitals

MetricMeasuresGood
LCP (Largest Contentful Paint)Loading< 2.5s
INP (Interaction to Next Paint)Responsiveness< 200ms
CLS (Cumulative Layout Shift)Visual stability< 0.1

How to improve each

  • LCP: optimize/preload the hero image, fast server/CDN, reduce render-blocking CSS/JS, fetchpriority="high".
  • INP: break up long JS tasks, defer non-critical JS, avoid heavy main-thread work, use web workers.
  • CLS: set width/height on images/embeds, reserve space for ads, avoid inserting content above existing content, use font-display: optional/swap.
Field vs lab: CrUX/Search Console = real-user field data (what ranks); Lighthouse/PageSpeed = lab data (for debugging).

JavaScript SEO & Rendering

Google renders JS, but it's slower and can miss content. Your rendering strategy directly affects indexing.

StrategySEOUse
CSR (client-side)Weakest — content needs JS executionDashboards, logged-in apps
SSR (server-side)Strong — HTML ready on requestDynamic, personalized pages
SSG (static)Strongest + fastestBlogs, docs, marketing
ISR (incremental)Strong — static + periodic revalidationE-commerce catalogs
CSR pitfalls: content behind JS may be delayed/missed; use SSR/SSG (Next.js/Nuxt) or dynamic rendering/prerendering for bots. Ensure links are real <a href> (not JS-only onClick).

🌍 React/Next SEO

Use Next.js with generateMetadata/next/head for per-page titles/canonicals, SSR/SSG for content pages, next/image for LCP/CLS, and route-level sitemaps.

Mobile & HTTPS

  • Mobile-first indexing: Google uses the mobile version for ranking — ensure parity of content/metadata.
  • Responsive design + correct viewport; tap targets & readable fonts.
  • HTTPS is a ranking signal and required for many APIs.

International SEO

hreflang for languages/regions
<link rel="alternate" hreflang="en-us" href="https://site.com/us/">
<link rel="alternate" hreflang="fr" href="https://site.com/fr/">
<link rel="alternate" hreflang="x-default" href="https://site.com/">
hreflang tells Google which language/region version to show; use consistent, reciprocal tags and a URL structure (subdirectory/subdomain/ccTLD).

Links & Authority (Off-Page)

  • Backlinks from authoritative sites are a top ranking factor (quality > quantity).
  • Internal linking spreads authority and helps crawling; use descriptive anchor text.
  • rel="nofollow"/sponsored/ugc for untrusted/paid/user links.
  • Avoid link schemes (buying links) — penalties.

Best Practices

  • Unique title + description per page; one h1; semantic structure.
  • Fast, mobile-first, HTTPS; pass Core Web Vitals.
  • Canonical URLs; clean, descriptive URLs; fix duplicates.
  • XML sitemap + robots.txt; submit to Search Console.
  • Structured data for eligible content.
  • SSR/SSG for content that must rank; real <a> links.
  • Quality content matching intent (E-E-A-T); earn backlinks.

Scenario-Based Questions

Scenario 1 — SPA content not indexed Rendering

Q: A React SPA's pages don't appear in Google. Why & how to fix?

Show approach

Content is client-rendered and may not be indexed reliably. Move to SSR/SSG (Next.js), ensure per-route meta/canonical, use real <a href> links and a sitemap, and verify with Search Console's URL Inspection (rendered HTML).

Scenario 2 — Traffic dropped after redesign Audit

Q: Organic traffic fell 40% post-launch.

Show approach

Check for accidental noindex/robots block, changed/missing canonicals, broken 301 redirects from old URLs, removed content, and slower Core Web Vitals. Compare Search Console coverage before/after.

Scenario 3 — Poor LCP on product pages CWV

Q: LCP is 5s on mobile.

Show approach

Optimize/compress + preload the hero image (fetchpriority="high"), use a CDN, remove render-blocking resources, lazy-load below-the-fold, and reduce server response time (TTFB).

Top Interview Questions

What are the three pillars of SEO?

Technical (crawl/index/speed), on-page (content/meta/structure), and off-page (backlinks/authority).

Crawling vs indexing vs ranking?

Crawling discovers URLs; indexing stores/understands content; ranking orders results for a query by relevance and authority signals.

robots.txt vs noindex?

robots.txt blocks crawling (page can still be indexed if linked); noindex keeps it out of the index but the page must be crawlable to see the tag.

What is a canonical tag?

It declares the preferred URL among duplicates, consolidating ranking signals to one page.

What are Core Web Vitals?

LCP (loading, <2.5s), INP (responsiveness, <200ms), CLS (visual stability, <0.1) — real-user UX metrics that affect ranking.

How do you improve LCP / CLS / INP?

LCP: optimize/preload hero, CDN, cut render-blocking. CLS: size media, reserve space, font-display. INP: break long tasks, defer JS, use workers.

CSR vs SSR vs SSG for SEO?

SSG is best (static HTML, fastest), SSR strong (HTML per request), CSR weakest (content needs JS). Use SSR/SSG for pages that must rank.

How does Google handle JavaScript?

It crawls, queues for rendering (headless Chromium), then indexes the rendered HTML — slower and imperfect, so critical content shouldn't rely solely on client JS.

What is structured data?

Schema.org markup (JSON-LD) describing content to enable rich results (ratings, FAQs, breadcrumbs).

What is mobile-first indexing?

Google predominantly uses the mobile version of a page for indexing/ranking — content and metadata must match desktop.

What is hreflang?

A tag telling Google which language/region version to serve, preventing duplicate-content issues across localized pages.

What is E-E-A-T?

Experience, Expertise, Authoritativeness, Trust — quality signals Google uses, especially for YMYL (your money/your life) topics.

301 vs 302 redirect for SEO?

301 (permanent) passes ranking signals to the new URL; 302 (temporary) keeps signals on the original — use 301 for permanent moves.

What causes duplicate content and how to fix?

URL params, http/https, www, trailing slashes, printer pages. Fix with canonicals, 301s, consistent internal linking, and parameter handling.

How do you diagnose an indexing problem?

Use Search Console URL Inspection (crawled/rendered/indexed state), check robots/noindex, canonicals, sitemaps, and server responses.
