Overview

Material UI (MUI) is a popular React component library implementing Google's Material Design. It gives you accessible, themeable, production-ready components out of the box.

  • Rich components — buttons, inputs, dialogs, tables, navigation, and more.
  • Theming — centralized design tokens (colors, typography, spacing).
  • Emotion-based styling — the sx prop & styled API.
  • Accessible — ARIA + keyboard behavior built in.
Interview importance: Theming, the sx prop vs styled, responsive breakpoints, and customization/overrides are the most-asked MUI topics.

Setup

Install + ThemeProvider
npm install @mui/material @emotion/react @emotion/styled @mui/icons-material

import { ThemeProvider, createTheme, CssBaseline } from "@mui/material";

const theme = createTheme();

<ThemeProvider theme={theme}>
  <CssBaseline />  // normalize + apply theme background
  <App />
</ThemeProvider>
CssBaseline resets browser defaults and applies theme background/typography — like Normalize for MUI.

Layout (Box, Grid, Stack)

Grid v2 + Stack + Box
<Grid container spacing={2}>
  <Grid size={{ xs: 12, md: 6 }}>...</Grid>
  <Grid size={{ xs: 12, md: 6 }}>...</Grid>
</Grid>

<Stack direction={{ xs: "column", sm: "row" }} spacing={2}>...</Stack>

<Box sx={{ p: 2, display: "flex", gap: 2 }}>...</Box>
Box is the styling primitive (adds the sx prop to any element). Stack handles 1D spacing; Grid handles 2D responsive layout; Container centers & max-widths content.

The sx Prop

A superset of CSS with theme-aware shorthands — the fastest way to style one-off components.

sx with theme tokens & breakpoints
<Box sx={{
  p: 2,                         // theme.spacing(2) = 16px
  color: "primary.main",        // theme palette token
  bgcolor: "grey.100",
  borderRadius: 2,
  display: { xs: "none", md: "block" }, // responsive
  "&:hover": { boxShadow: 3 }         // pseudo-selectors
}}/>
Trade-off: sx is convenient but generates styles at runtime. For heavily reused components, prefer styled or theme variants.

styled API & Overrides

styled component
import { styled } from "@mui/material/styles";

const Fancy = styled(Button)(({ theme }) => ({
  borderRadius: 999,
  padding: theme.spacing(1, 3),
  background: theme.palette.primary.main
}));

sx prop

  • Inline, quick, one-offs
  • Runtime cost per render

styled

  • Reusable styled components
  • Better for repeated use
Legacy note: makeStyles/withStyles (JSS) are deprecated (MUI v5+ uses Emotion). Migrate to sx/styled.

Theming

createTheme: palette, typography, components
const theme = createTheme({
  palette: {
    primary: { main: "#7c3aed" },
    secondary: { main: "#f59e0b" }
  },
  typography: { fontFamily: "Inter, sans-serif", button: { textTransform: "none" } },
  shape: { borderRadius: 10 },
  components: {
    MuiButton: {
      defaultProps: { disableElevation: true },
      styleOverrides: { root: { fontWeight: 600 } },
      variants: [{ props: { variant: "pill" }, style: { borderRadius: 999 } }]
    }
  }
});
  • defaultProps — set default props for a component globally.
  • styleOverrides — override component slot styles.
  • variants — define custom variants reusable via variant="pill".
  • Access anywhere with useTheme().

Dark Mode

Toggle color mode
const [mode, setMode] = useState("light");
const theme = useMemo(() => createTheme({ palette: { mode } }), [mode]);

// respect OS preference
const prefersDark = useMediaQuery("(prefers-color-scheme: dark)");
MUI computes appropriate text/background colors per mode automatically when you set palette.mode. Persist the choice in localStorage.

Performance

  • Import per-path is handled automatically in v5 (named imports tree-shake) — avoid deep default imports.
  • Prefer styled/theme variants over many inline sx for hot components.
  • Memoize the theme object (don't recreate on every render).
  • Virtualize large lists/tables (DataGrid, react-window).
  • Lazy-load heavy components (Dialog content, DataGrid).
Bundle size: Emotion + MUI adds weight. Tree-shake with named imports, code-split routes, and only import icons you use (@mui/icons-material/Save).

Accessibility

  • Components ship with ARIA + keyboard behavior (Dialog focus trap, Menu arrow keys).
  • Always label icon-only buttons (aria-label) and inputs (label).
  • Don't remove focus outlines; respect theme contrast.
  • Use semantic components (Button, not clickable Box).

Best Practices

  • Centralize design in the theme; avoid hardcoded colors/spacing.
  • sx for one-offs, styled/variants for reuse.
  • Use theme tokens (primary.main, spacing) not magic numbers.
  • Wrap the app once in ThemeProvider + CssBaseline.
  • Tree-shake imports; code-split; memoize the theme.
  • Label icon buttons/inputs; keep accessibility intact.

Scenario-Based Questions

Scenario 1 — Rebrand all buttons & colors Theming

Q: The whole app must switch to a new brand color and rounded buttons.

Show approach

Change it in one place: update palette.primary and components.MuiButton.styleOverrides/defaultProps in createTheme. Because components read theme tokens, the whole app updates.

Scenario 2 — Large table is janky Performance

Q: A 10k-row MUI Table freezes the UI.

Show approach

Use MUI X DataGrid (virtualized) or react-window; server-side pagination/sorting; memoize rows; avoid heavy sx per cell.

Scenario 3 — Bundle too big Bundle

Q: The app ships a huge JS bundle after adding MUI.

Show approach

Use named imports (tree-shaking), import icons by path, code-split routes/heavy components (Dialog, DataGrid) with React.lazy, and analyze with a bundle analyzer.

Top Interview Questions

What is Material UI?

A React component library implementing Material Design, with theming, accessible components, and Emotion-based styling (sx/styled).

sx prop vs styled?

sx is inline, theme-aware styling for one-offs (runtime cost); styled creates reusable styled components — better for repeated use.

How does theming work?

createTheme defines palette/typography/spacing/component overrides; ThemeProvider supplies it; components and sx read tokens so a single change re-skins the app.

What is CssBaseline?

A component that normalizes browser styles and applies the theme's background/typography — MUI's version of a CSS reset.

How do you make MUI responsive?

Use responsive object values in sx/props ({ xs, sm, md }), the Grid/Stack breakpoints, and useMediaQuery for logic.

How do you override a component's styles globally?

Via theme.components.MuiX.styleOverrides (and defaultProps/variants) in createTheme.

How do you implement dark mode?

Set palette.mode to "dark" in the theme (toggle state, memoized theme); optionally follow prefers-color-scheme and persist the choice.

Box vs Container vs Stack vs Grid?

Box = styling primitive; Container = centered max-width wrapper; Stack = 1D spacing; Grid = 2D responsive layout.

Why is makeStyles deprecated?

MUI v5 moved from JSS to Emotion; use sx and styled instead of makeStyles/withStyles.

How do you reduce MUI bundle size?

Named imports (tree-shaking), import icons by path, code-split routes/heavy components, and analyze the bundle.

How do you integrate forms?

Use React Hook Form/Formik; wrap non-native components (Select, Autocomplete) in a Controller, and drive error/helperText from validation state.

How do you create a custom variant?

Add it under components.MuiButton.variants in the theme, then use variant="pill". Declare the type in TS for autocomplete.

Table vs DataGrid?

Table is a simple presentational component; MUI X DataGrid adds sorting, filtering, pagination, and virtualization for large datasets.

How do you access the theme in a component?

useTheme() hook, or the theme callback in sx/styled.

Is MUI accessible by default?

Largely yes — components include ARIA and keyboard support — but you must still label icon buttons/inputs and preserve focus/contrast.
