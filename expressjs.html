Overview

Express is a minimal, unopinionated web framework for Node.js. It provides routing, middleware, and helpers for building HTTP APIs and web servers.

  • Middleware pipeline — the core concept; requests flow through ordered functions.
  • Routing — map HTTP method + path to handlers.
  • Unopinionated — you choose structure, DB, auth, etc.
  • Huge ecosystem — helmet, cors, morgan, multer, passport.
Interview importance: Middleware order, error handling, and auth/security are the most-asked Express topics.

Middleware

Middleware are functions (req, res, next) that run in order. They can modify req/res, end the response, or call next() to pass control.

Custom + order matters
// Application-level
app.use((req, res, next) => { req.time = Date.now(); next(); });

// Route-level (auth guard)
function auth(req, res, next) {
  if (!req.headers.authorization) return res.sendStatus(401);
  next();
}
app.get("/secure", auth, handler);
TypeExample
Applicationapp.use(fn)
Routerrouter.use(fn)
Routeapp.get(path, mw, handler)
Built-inexpress.json(), express.static()
Error(err, req, res, next) — 4 args
Order is everything: body parsers must come before routes; error handlers must be registered last; a middleware that forgets next() hangs the request.

MVC & Services

Layered structure
src/
  routes/        // map path -> controller
  controllers/   // parse req, call service, send res
  services/      // business logic
  models/        // DB schema/queries
  middleware/    // auth, validation, errors
  utils/  config/
Controller → Service split
// controller
exports.create = async (req, res, next) => {
  try { res.status(201).json(await userService.create(req.body)); }
  catch (e) { next(e); } // forward to error middleware
};
Why layers: controllers stay thin (HTTP concerns), services hold reusable business logic (testable, framework-agnostic), models isolate data access.

Validation

Zod validation middleware
const schema = z.object({ email: z.string().email(), age: z.number().min(0) });

const validate = (schema) => (req, res, next) => {
  const r = schema.safeParse(req.body);
  if (!r.success) return res.status(400).json(r.error.issues);
  req.body = r.data; next();
};
Never trust input. Validate & sanitize at the boundary (zod/joi/express-validator) to prevent injection and bad data.

Error Handling

Async wrapper + central handler
// Wrap async handlers so rejections reach the handler
const ah = fn => (req, res, next) => Promise.resolve(fn(req, res, next)).catch(next);

// 404 handler (after routes)
app.use((req, res) => res.status(404).json({ error: "Not found" }));

// Central error handler (LAST, 4 args)
app.use((err, req, res, next) => {
  const status = err.status || 500;
  logger.error({ err, path: req.path });
  res.status(status).json({ error: err.message });
});
Pattern: Throw typed errors (a custom AppError with a status) from services; one central handler formats them consistently.

Auth & Authorization

JWT auth + role guard
function authenticate(req, res, next) {
  const token = req.headers.authorization?.split(" ")[1];
  try { req.user = jwt.verify(token, process.env.SECRET); next(); }
  catch { res.sendStatus(401); }
}
const authorize = (...roles) => (req, res, next) =>
  roles.includes(req.user.role) ? next() : res.sendStatus(403);

app.delete("/users/:id", authenticate, authorize("admin"), ctrl.remove);
401 vs 403: 401 = not authenticated (who are you?); 403 = authenticated but not allowed (you can't do this).

Scenario-Based Questions

Scenario 1 — Requests hang forever Middleware

Q: Some routes never respond. What's likely wrong?

Show approach

A middleware forgot to call next() or send a response. Check the pipeline order and ensure each middleware either responds or calls next().

Scenario 2 — Async errors return 200 with empty body Errors

Q: Thrown errors in async handlers aren't caught.

Show approach

Express (v4) doesn't catch async rejections automatically. Wrap handlers (asyncHandler) so errors go to next(err) and the central error middleware. (Express 5 handles this natively.)

Scenario 3 — CORS errors in the browser Security

Q: Frontend calls fail with a CORS error.

Show approach

Configure the cors middleware with the correct origin allow-list, allowed methods/headers, and credentials: true if using cookies. Handle preflight (OPTIONS) automatically.

Production Issues

"Cannot set headers after they are sent" Bug

Report: Server logs this error intermittently.

Show fix

Cause: Sending a response twice (e.g. missing return before res.json, then continuing). Fix: return after sending, and don't call next() after responding.

Body is undefined on POST Bug

Report: req.body is undefined.

Show fix

Cause: Missing/late body parser. Fix: app.use(express.json()) before routes, and ensure the client sends Content-Type: application/json.

Top Interview Questions

What is middleware in Express?

Functions (req, res, next) that run in order in the request pipeline; they can modify req/res, end the response, or call next() to continue.

How does error-handling middleware differ?

It takes four args (err, req, res, next) and must be registered last. Express routes errors (from next(err) or sync throws) to it.

app.use vs app.get?

app.use mounts middleware for all methods/paths (prefix match); app.get matches a specific method + exact path.

How do you handle async errors?

Wrap handlers to catch promise rejections and forward via next(err) (Express 4), or use Express 5 which awaits handlers. Add a central error middleware.

Why does middleware order matter?

Middleware runs top-to-bottom: parsers before routes, auth before protected routes, error handler last. Wrong order breaks parsing/auth/errors.

401 vs 403?

401 = unauthenticated (missing/invalid credentials); 403 = authenticated but not permitted.

How do you structure a large Express app?

Layered/MVC: routes → controllers → services → models, plus middleware/config/utils. Modular routers per resource with API versioning.

How do you secure an Express API?

helmet, CORS allow-list, input validation, JWT/bcrypt, rate limiting, body size limits, HTTPS, secrets in env, disable x-powered-by, npm audit.

express.json vs body-parser?

body-parser was separate; its functionality is now built into Express as express.json()/express.urlencoded().

What is a Router?

A mini, mountable app for grouping routes/middleware by resource — express.Router() mounted at a base path.

How do route params and query differ?

req.params come from the path (/users/:id); req.query from the query string (?page=2).

How do you serve static files?

app.use(express.static("public")); put a reverse proxy/CDN in front in production.

How do you implement rate limiting?

express-rate-limit per IP/user; back it with Redis for multi-instance deployments.

What causes "headers already sent"?

Responding more than once. Always return after sending and don't continue the pipeline afterward.

How do you do graceful shutdown?

On SIGTERM: server.close() to stop new connections, drain in-flight requests, close DB, then exit.
