Overview

React is a declarative, component-based library for building UIs. You describe what the UI should look like for a given state, and React efficiently updates the DOM.

Core ideas

  • Declarative — UI = f(state). Describe the result, not the steps.
  • Component-based — compose small, reusable pieces.
  • Virtual DOM + reconciliation — diff and update only what changed.
  • Unidirectional data flow — props down, events up.
Interview importance: React is the most in-demand frontend skill. Hooks, rendering behavior, performance, and state management dominate interviews.

JSX

JSX is syntax sugar for React.createElement. It lets you write HTML-like markup in JavaScript.

JSX compiles to createElement
// JSX
const el = <h1 className="title">Hello {name}</h1>;

// compiles to
const el = React.createElement("h1", { className: "title" }, "Hello ", name);
  • Use className (not class), htmlFor (not for).
  • Expressions in { }; must return a single root (use fragments <>...</>).
  • JSX prevents XSS by escaping values by default.

Components & Props

Function component + props
function Greeting({ name, isVip = false }) {
  return <p>Hello {name}{isVip && " ⭐"}</p>;
}
<Greeting name="Aria" isVip />
  • Props are read-only — never mutate them (one-way data flow).
  • children prop enables composition: <Card>...</Card>.
  • Prefer function components + hooks over classes.

State & Events

Counter with state
function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}
State updates are async & batched. Use the functional updater setCount(c => c + 1) when the next value depends on the previous.

Rendering & the Virtual DOM

React keeps a lightweight copy of the DOM (Virtual DOM). On state/prop change it re-renders, diffs the new tree against the old (reconciliation), and applies the minimal real-DOM updates.

  • A component re-renders when its state or props change, or its parent re-renders.
  • Keys help React match list items across renders — use stable, unique keys (not index for dynamic lists).
  • Fiber architecture enables interruptible, prioritized rendering (concurrent features).
Reconciliation heuristics: different element types → replace subtree; same type → update props; lists → matched by key.

Conditional Rendering & Lists

Conditionals & lists with keys
{isLoading ? <Spinner/> : <List items={items}/>}
{error && <Error msg={error}/>}

{users.map(u => <li key={u.id}>{u.name}</li>)}
Don't use array index as key for lists that reorder/insert/delete — it causes wrong state association and subtle bugs.

Forms — Controlled vs Uncontrolled

Controlled

React state is the source of truth.

controlled input
const [v, setV] = useState("");
<input value={v} onChange={e => setV(e.target.value)}/>

Uncontrolled

DOM holds the value; read via ref.

uncontrolled input
const ref = useRef();
<input ref={ref} defaultValue="hi"/>
// read: ref.current.value
Rule: Prefer controlled for validation/live updates; uncontrolled (or libraries like React Hook Form) for large forms to reduce re-renders.

Composition & Lifting State

Share state by lifting it to the closest common ancestor and passing it down as props. Compose UI with children and specialized components rather than inheritance.

Lifting state up
function Parent() {
  const [q, setQ] = useState("");
  return <><Search value={q} onChange={setQ}/><Results query={q}/></>;
}

Context API

Context shares data (theme, auth, locale) across the tree without prop-drilling.

Create, provide, consume
const ThemeCtx = createContext("light");

<ThemeCtx.Provider value={theme}><App/></ThemeCtx.Provider>

const theme = useContext(ThemeCtx); // consume anywhere below
Perf caveat: Every consumer re-renders when the context value changes. Split contexts, memoize the value, or use a state library for high-frequency updates.

useState

What: declares local state. Why: trigger re-render on change. How: returns [value, setValue].

Lazy init & functional updates
const [state, setState] = useState(() => expensiveInit()); // lazy init runs once
setState(prev => ({ ...prev, count: prev.count + 1 }));    // merge objects yourself
Mistakes: mutating state directly; relying on state immediately after setState (it's async); forgetting objects/arrays need new references.

useEffect

What: run side effects (data fetch, subscriptions, timers) after render. Why: sync component with external systems.

Effect with cleanup & deps
useEffect(() => {
  const id = setInterval(tick, 1000);
  return () => clearInterval(id); // cleanup on unmount / before re-run
}, [dep]); // [] = mount only; none = every render
DepsRuns
NoneAfter every render
[]Once on mount
[a, b]When a or b changes
Mistakes: missing dependencies (stale closures), infinite loops (setting state without deps), not cleaning up subscriptions/timers.

useRef

What: a mutable box (.current) that persists across renders without causing re-renders. Use: DOM access, storing timers/previous values.

DOM ref & mutable value
const inputRef = useRef(null);
useEffect(() => inputRef.current?.focus(), []);
<input ref={inputRef}/>

useMemo & useCallback

useMemo memoizes a value; useCallback memoizes a function. Both avoid unnecessary recomputation/re-creation across renders.

Memoization
const sorted = useMemo(() => items.sort(cmp), [items]);   // expensive calc
const onClick = useCallback(() => doX(id), [id]);     // stable fn for memo children
Don't over-memoize. These have a cost. Use when a computation is expensive or a stable reference is needed by a memoized child / effect dependency.

useReducer

For complex state logic with many transitions — a Redux-like reducer local to a component.

Reducer pattern
function reducer(state, action) {
  switch (action.type) {
    case "inc": return { count: state.count + 1 };
    default: return state;
  }
}
const [state, dispatch] = useReducer(reducer, { count: 0 });
dispatch({ type: "inc" });

Custom Hooks

A custom hook is a function starting with use that composes built-in hooks to share stateful logic.

useFetch
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  useEffect(() => {
    const ctrl = new AbortController();
    fetch(url, { signal: ctrl.signal })
      .then(r => r.json()).then(setData)
      .finally(() => setLoading(false));
    return () => ctrl.abort();
  }, [url]);
  return { data, loading };
}
useDebounce & useLocalStorage
function useDebounce(value, delay = 300) {
  const [v, setV] = useState(value);
  useEffect(() => {
    const id = setTimeout(() => setV(value), delay);
    return () => clearTimeout(id);
  }, [value, delay]);
  return v;
}

function useLocalStorage(key, init) {
  const [val, setVal] = useState(() =>
    JSON.parse(localStorage.getItem(key)) ?? init);
  useEffect(() => localStorage.setItem(key, JSON.stringify(val)), [key, val]);
  return [val, setVal];
}
Common custom hooks: usePrevious, useToggle, useOnClickOutside, useMediaQuery, useIntersectionObserver, useEventListener.

Refs, forwardRef & useImperativeHandle

Expose an imperative API
const Input = forwardRef((props, ref) => {
  const inner = useRef();
  useImperativeHandle(ref, () => ({ focus: () => inner.current.focus() }));
  return <input ref={inner} {...props}/>;
});

Portals

Render children into a DOM node outside the parent hierarchy — ideal for modals, tooltips, toasts (avoids overflow/z-index issues).

createPortal
return createPortal(
  <div className="modal">{children}</div>,

Error Boundaries

Class components that catch render errors in their subtree and show a fallback. (No hook equivalent yet — use a class or a library.)

ErrorBoundary
class ErrorBoundary extends React.Component {
  state = { hasError: false };
  static getDerivedStateFromError() { return { hasError: true }; }
  componentDidCatch(err, info) { logToService(err, info); }
  render() { return this.state.hasError ? <Fallback/> : this.props.children; }
}

Lazy Loading & Suspense

Code splitting
const Dashboard = lazy(() => import("./Dashboard"));

<Suspense fallback={<Spinner/>}>
  <Dashboard/>
</Suspense>
Benefit: Splits the bundle so routes/heavy components load on demand — faster initial load.

Redux Toolkit

Redux centralizes global state. Redux Toolkit (RTK) is the official, boilerplate-free way — slices, immutable updates via Immer, and a preconfigured store.

createSlice + store
const counter = createSlice({
  name: "counter",
  initialState: { value: 0 },
  reducers: {
    increment: s => { s.value++; },        // Immer: "mutate" safely
    addBy: (s, a) => { s.value += a.payload; }
  }
});
export const { increment, addBy } = counter.actions;

const store = configureStore({ reducer: { counter: counter.reducer } });
useSelector + useDispatch
const value = useSelector(s => s.counter.value);
const dispatch = useDispatch();
dispatch(increment());
Data flow: Component → dispatch(action) → reducer computes new state → subscribed components re-render. Single source of truth, predictable, time-travel debuggable.

RTK Query & React Query

Server-state libraries handle fetching, caching, deduping, and revalidation — replacing hand-written useEffect fetch logic.

RTK Query

createApi
const api = createApi({
  baseQuery: fetchBaseQuery({ baseUrl: "/api" }),
  endpoints: b => ({
    getUsers: b.query(() => "users")
  })
});
const { data, isLoading } = useGetUsersQuery();

React Query (TanStack)

useQuery
const { data, isLoading } = useQuery({
  queryKey: ["users"],
  queryFn: () => axios.get("/api/users").then(r => r.data)
});
When to use what: Context for low-frequency global state (theme/auth); Redux for complex client state; RTK Query / React Query for server state (most data fetching).

Auth & Protected Routes

ProtectedRoute + Axios interceptor
function Protected({ children }) {
  const { user } = useAuth();
  return user ? children : <Navigate to="/login" replace/>;
}

// attach token to every request
axios.interceptors.request.use(cfg => {
  cfg.headers.Authorization = `Bearer ${getToken()}`;
  return cfg;
});
Security: Prefer HttpOnly cookies over localStorage for tokens (XSS-safe), and implement refresh-token rotation.

Architecture

Feature-based folder structure
src/
  app/            // store, routes, providers
  features/
    auth/         // components, hooks, slice, api, tests
    products/
  components/     // shared UI
  hooks/  lib/  utils/  styles/
  • Feature-based beats type-based at scale (colocate by domain).
  • Atomic design for the component library (atoms → molecules → organisms).
  • Micro-frontends (Module Federation) split a large app across teams.
  • Next.js adds SSR/SSG/RSC, routing, and data fetching on top of React.

Testing

React Testing Library
test("increments", async () => {
  render(<Counter/>);
  await userEvent.click(screen.getByRole("button"));

Accessibility in React

  • Use semantic elements + ARIA only when needed; label controls (htmlFor/aria-label).
  • Manage focus on route change & modal open/close; trap focus in dialogs.
  • Use useId for associating labels/inputs (SSR-safe).
  • Ensure keyboard operability and visible focus states.

Scenario-Based Questions

Scenario 1 — 5000-row table is slow Performance

Q: A list of 5000 rows lags on every keystroke. How do you optimize?

Show approach

Virtualize (react-window) to render only visible rows; React.memo row components; useMemo the filtered/sorted data; stable keys; move the input state so the whole list doesn't re-render; consider useDeferredValue for the filter.

Scenario 2 — Stale closure in useEffect Hooks

Q: A timer logs an old state value. Why, and how do you fix it?

Show approach

The effect captured the initial state (stale closure). Fix by adding the value to deps, using the functional updater setX(x => ...), or storing the latest value in a ref.

Scenario 3 — Prop drilling hell State

Q: A value is passed through 6 layers of components. How do you improve it?

Show approach

Lift to Context for low-frequency global data, or a state library (Redux/Zustand) for complex/high-frequency state. Compose with children to avoid unnecessary intermediaries.

Production Issues

Infinite re-render loop Bug

Report: Page freezes; "Maximum update depth exceeded".

Show fix

Cause: calling setState during render, or an object/array in effect deps recreated each render. Fix: move updates into effects/handlers, memoize deps, and correct the dependency array.

Memory leak: setState after unmount Bug

Report: Warning about updating state on an unmounted component.

Show fix

Cause: async resolves after unmount. Fix: abort fetches with AbortController and clean up subscriptions/timers in the effect return.

Top Interview Questions

What is the Virtual DOM and reconciliation?

An in-memory tree of elements. On update, React builds a new tree, diffs it against the old (reconciliation) using type + key heuristics, and applies minimal real-DOM changes.

Why are keys important in lists?

Keys let React match elements across renders to preserve state and minimize DOM ops. Use stable unique IDs; index keys break on reorder/insert/delete.

useEffect vs useLayoutEffect?

useEffect runs asynchronously after paint; useLayoutEffect runs synchronously before paint (use to measure/adjust DOM to avoid flicker).

What causes a component to re-render?

State change, prop change, parent re-render, or context value change. Use React.memo/useMemo/useCallback to prevent unnecessary ones.

Controlled vs uncontrolled components?

Controlled: value driven by React state (predictable, validatable). Uncontrolled: DOM holds the value, read via ref (less re-render, simpler for big forms).

What is a custom hook?

A reusable function starting with use that composes built-in hooks to share stateful logic between components without HOCs/render props.

Context vs Redux?

Context passes data without prop-drilling but re-renders all consumers on change — best for low-frequency global state. Redux offers structured, debuggable, high-frequency state management with middleware and selectors.

What does React.memo do?

Memoizes a component so it skips re-render when props are shallow-equal. Pair with useCallback/useMemo so prop references stay stable.

Explain the rules of hooks.

Only call hooks at the top level (not in loops/conditions) and only from React functions/custom hooks — so React can track hook order reliably.

How does React Router handle protected routes?

Wrap routes in a component that checks auth and renders children or redirects with <Navigate>. Combine with loaders for data-aware guarding.

What is prop drilling and how to avoid it?

Passing props through many intermediate components. Avoid with Context, composition (children), or a state library.

What are error boundaries?

Class components that catch render/lifecycle errors in their subtree via getDerivedStateFromError/componentDidCatch and show a fallback UI.

Server state vs client state?

Client state is UI/local (form inputs, toggles). Server state is remote data with caching/staleness concerns — handled best by React Query/RTK Query.

What is Suspense?

A mechanism to declaratively show a fallback while lazy components or data are loading, enabling code splitting and (with frameworks) streaming.

Class vs function components?

Function components with hooks are the modern standard — less boilerplate, easier logic reuse. Classes remain only for error boundaries.
