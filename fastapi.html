Overview

FastAPI is a modern, high-performance Python web framework built on Starlette (ASGI) and Pydantic. It gives you type-hint-driven validation, automatic OpenAPI docs, and async support.

  • Fast — on par with Node/Go thanks to ASGI + Starlette + Uvicorn.
  • Type-safe — Python type hints drive validation & serialization.
  • Auto docs — Swagger UI (/docs) & ReDoc (/redoc) for free.
  • DI system — clean, testable dependency injection.
Interview importance: Pydantic validation, dependency injection, async vs sync, and OAuth2/JWT security are the most-asked FastAPI topics.

Setup & First App

main.py

async def root():

Path & Query Parameters

Typed params with validation
from fastapi import Query, Path

@app.get("/items/{item_id}")
async def get_item(
    item_id: int = Path(ge=1),               # path param, >= 1
    q: str | None = Query(None, max_length=50), # optional query
    limit: int = Query(10, le=100)
):
    return {"item_id": item_id, "q": q, "limit": limit}
Type hints = validation. FastAPI parses, validates, and coerces params from the type annotation, returning a clear 422 on bad input.

Pydantic Models (v2)

Request/response models
from pydantic import BaseModel, Field, EmailStr

class UserCreate(BaseModel):
    name: str = Field(min_length=1)
    email: EmailStr
    age: int = Field(ge=0, le=120)

class UserOut(BaseModel):
    id: int
    name: str
    email: EmailStr
    model_config = {"from_attributes": True}  # ORM mode

@app.post("/users", response_model=UserOut, status_code=201)
async def create(user: UserCreate):
    return save(user)
FeaturePurpose
BaseModelSchema + validation + serialization
FieldConstraints, defaults, metadata
response_modelShape & filter the response (hide secrets)
from_attributesBuild from ORM objects (v1: orm_mode)

Validation

Custom validators (Pydantic v2)
from pydantic import field_validator, model_validator

class Signup(BaseModel):
    password: str
    confirm: str

    @field_validator("password")
    @classmethod
    def strong(cls, v):
        if len(v) < 8: raise ValueError("too short")
        return v

    @model_validator(mode="after")
    def match(self):
        if self.password != self.confirm: raise ValueError("mismatch")
        return self
Automatic 422: Validation failures return a structured 422 with field-level error details — no manual checking.

Dependency Injection

FastAPI's DI system injects shared logic (DB sessions, auth, pagination) into routes — declarative, testable, and cached per request.

Depends + yield for cleanup
from fastapi import Depends

def get_db():
    db = SessionLocal()
    try: yield db          # injected value
    finally: db.close()   # cleanup after response

def pagination(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

@app.get("/users")
async def list_users(db=Depends(get_db), page=Depends(pagination)):
    return query(db, **page)
  • Sub-dependencies — dependencies can depend on other dependencies.
  • Caching — the same dependency is resolved once per request.
  • Overrides — app.dependency_overrides makes testing trivial.
  • Class dependencies — a callable class works as a dependency too.

Async & Concurrency

async def vs def
@app.get("/a")
async def a():
    await async_db.fetch()   # non-blocking on the event loop

@app.get("/b")
def b():
    return blocking_lib.call()  # runs in a threadpool automatically
Golden rule: Never call a blocking function inside async def — it blocks the whole event loop. Use an async library, or a regular def (FastAPI runs sync routes in a thread pool), or run_in_executor.
HandlerRuns
async defOn the event loop — use only non-blocking awaits
defIn an external threadpool — safe for blocking I/O

ASGI Internals

Understanding the stack helps in senior interviews.

LayerRole
Uvicorn/HypercornASGI server — runs the event loop, handles HTTP/WebSocket
StarletteASGI toolkit — routing, middleware, requests/responses
FastAPIAdds validation, DI, OpenAPI on top of Starlette
PydanticData validation & serialization
  • ASGI (Async Server Gateway Interface) is the async successor to WSGI — supports async, WebSockets, and long-lived connections.
  • Workers: run multiple Uvicorn workers (--workers) or Gunicorn with the Uvicorn worker class to use all cores.
  • Lifespan events (@asynccontextmanager) start/stop resources (DB pools) at boot/shutdown.
Lifespan startup/shutdown
from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app):
    await db.connect()   # startup
    yield
    await db.disconnect() # shutdown

app = FastAPI(lifespan=lifespan)

Middleware

Custom + CORS
from fastapi.middleware.cors import CORSMiddleware
app.add_middleware(CORSMiddleware, allow_origins=["https://app.com"])

@app.middleware("http")
async def timing(request, call_next):
    start = time.time()
    response = await call_next(request)
    response.headers["X-Time"] = str(time.time() - start)
    return response

OAuth2 & JWT

OAuth2 password flow + JWT
from fastapi.security import OAuth2PasswordBearer
oauth2 = OAuth2PasswordBearer(tokenUrl="token")

async def current_user(token: str = Depends(oauth2)):
    try:
        payload = jwt.decode(token, SECRET, algorithms=["HS256"])
    except JWTError:
        raise HTTPException(401, "Invalid token")
    return get_user(payload["sub"])

@app.get("/me")
async def me(user=Depends(current_user)):
    return user
Scopes: OAuth2 scopes + a Security() dependency implement fine-grained, role/permission-based authorization.

Background Tasks

BackgroundTasks vs Celery
from fastapi import BackgroundTasks

@app.post("/signup")
async def signup(user: UserCreate, tasks: BackgroundTasks):
    save(user)
    tasks.add_task(send_welcome_email, user.email) # after response
    return {"ok": True}
When to use what: BackgroundTasks for quick fire-and-forget after the response; Celery/RQ/ARQ + Redis for durable, retryable, distributed jobs.

Redis cache-aside
async def get_product(id: int):

Scenario-Based Questions

Scenario 1 — One slow endpoint stalls all requests Async

Q: An async def route calls a blocking DB driver and the whole app freezes under load.

Show approach

A blocking call inside async def blocks the event loop. Use an async driver (asyncpg), switch the route to a normal def (runs in a threadpool), or wrap the call in run_in_executor.

Scenario 2 — Secrets leaking in responses Pydantic

Q: The API returns password hashes.

Show approach

Use a separate response_model (e.g. UserOut) that only includes safe fields — FastAPI filters the output to that schema.

Scenario 3 — N+1 queries in a list endpoint DB

Q: Listing orders with their items is slow.

Show approach

Eager-load relations (selectinload/joinedload) instead of lazy loading per row, add indexes, and paginate.

Production Issues

422 errors clients don't understand Validation

Report: Clients get cryptic 422s.

Show fix

Add a custom RequestValidationError handler that reshapes Pydantic errors into your API's error format, and document expected schemas in OpenAPI.

Event loop blocked warnings Async

Report: High latency; "coroutine was never awaited" / blocked loop.

Show fix

Find sync/blocking calls in async paths (CPU work, blocking I/O, time.sleep). Move to async libs or threadpools; never forget await.

Top Interview Questions

What makes FastAPI fast?

It's built on Starlette (ASGI) + Uvicorn for async I/O, and uses Pydantic (compiled in v2) for fast validation. Async lets it handle many concurrent connections.

async def vs def routes?

async def runs on the event loop (use non-blocking awaits only); plain def runs in a threadpool, so it's safe for blocking I/O.

How does dependency injection work?

Depends(fn) resolves and injects the result; dependencies can nest, are cached per request, support yield for cleanup, and can be overridden in tests.

What is Pydantic's role?

It validates and serializes data from type hints — request bodies, query/path params, and response models — returning structured 422 errors automatically.

What is response_model for?

It defines and filters the output schema — hiding internal/sensitive fields and generating accurate OpenAPI docs.

What is ASGI vs WSGI?

WSGI is synchronous (Flask/Django classic); ASGI is asynchronous, supporting async views, WebSockets, and long-lived connections — FastAPI is ASGI.

How do you secure a FastAPI app?

OAuth2 password flow + JWT via OAuth2PasswordBearer, scopes for authorization, HTTPS, CORS allow-list, input validation, secrets in env.

BackgroundTasks vs Celery?

BackgroundTasks run in-process after the response (quick, non-durable). Celery/ARQ + Redis provide durable, retryable, distributed job processing.

How do you handle DB sessions?

A dependency with yield opens a session and closes it after the response — one session per request, cleaned up reliably.

Pydantic v1 vs v2 differences?

v2 is much faster (Rust core), uses model_config/from_attributes, field_validator/model_validator, and model_dump() instead of .dict().

How do you run FastAPI in production?

Gunicorn with Uvicorn workers (multiple processes for cores) behind nginx/traefik for TLS and load balancing, in Docker/Kubernetes with health probes.

What is a lifespan event?

An async context manager that runs startup (open DB pool) and shutdown (close resources) logic around the app's lifetime.

How do you test FastAPI?

TestClient (httpx) for requests + dependency_overrides to swap real DB/auth with fakes; pytest for the suite.

How do you validate nested/complex bodies?

Compose Pydantic models (models within models, lists of models); FastAPI validates the whole structure and documents it in OpenAPI.

How do you add CORS?

app.add_middleware(CORSMiddleware, allow_origins=[...]) with explicit origins, methods, headers, and credentials as needed.
